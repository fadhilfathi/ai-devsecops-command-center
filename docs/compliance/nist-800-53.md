# NIST SP 800-53 Rev. 5 — Mapping

This document maps the AI-DevSecOps Command Center to NIST SP 800-53
Rev. 5. The Command Center targets **Moderate** baseline compliance for
its core platform and supports customers operating at Moderate and
**High** baselines.

Each control is evaluated for:

- **Applicability** — Platform, Customer, Both, or N/A.
- **Implementation status** — see [legend](./README.md#compliance-status-legend).
- **Owner** — teammate agent responsible.
- **Implementation summary** — how the control is met.
- **Evidence** — pointer to the artefact that demonstrates operation.

> Companion documents:
> - CIS v8 mapping: [`cis-controls-v8.md`](./cis-controls-v8.md)
> - Cross-framework matrix: [`compliance-matrix.md`](./compliance-matrix.md)
> - Audit logging requirements: [`audit-logging.md`](./audit-logging.md)
> - Evidence methodology: [`evidence-collection.md`](./evidence-collection.md)

## Baseline tailoring

The Command Center's v1 platform baseline is **NIST 800-53 Moderate**.
The following control families are **scoped out** with justification:

- **PE (Physical and Environmental Protection)** — N/A; the platform is
  SaaS hosted in a FedRAMP-authorized cloud. Controls inherited from the
  cloud provider's authorization package.
- **MA (Maintenance)** — partially scoped; only the non-physical portions
  (e.g., MA-2 controlled maintenance via GitOps) are addressed by the
  Command Center directly. The rest is inherited from the cloud.
- **PS (Personnel Security)** — partially scoped; the Command Center
  documents its own personnel security program in
  `evidence-collection.md` §Personnel. Customer-side PS is the customer's
  responsibility.

## Acronyms

- **PM** = Program Management
- **RA** = Risk Assessment
- **CA** = Assessment, Authorization, and Monitoring
- **SI** = System and Services Acquisition
- **SA** = System and Services Acquisition (NIST uses SA for both)
- **SR** = Supply Chain Risk Management
